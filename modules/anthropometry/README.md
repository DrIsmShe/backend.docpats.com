# Anthropometry Module

Модуль для антропометрических измерений в пластической хирургии.

## Назначение

Позволяет хирургам:

- Загружать фотографии пациентов в стандартизированных проекциях
- Размечать анатомические точки вручную
- Получать автоматические вычисления углов, расстояний и пропорций
- Хранить историю измерений (версии аннотаций)
- Сравнивать результаты до/после операции

## Структура модуля

\```
modules/anthropometry/

├── models/ — Mongoose-схемы

## Зависимости

- `mongoose` — ODM для MongoDB
- `common/utils/crypto.js` — шифрование PHI-полей
- Модели `User`, `DoctorProfile`, `NewPatientPolyclinic`, `DoctorPrivatePatient` (через ref)

## Безопасность

- Все PHI-поля шифруются AES-256-CBC через `common/utils/crypto.js`
- Доступ к случаям ограничен middleware `caseOwnership.js`
- Все действия логируются в `AnthropometryAuditLog` (HIPAA § 164.312(b))
- Rate limiting на критических эндпоинтах

## API

API будет доступен под префиксом `/api/anthropometry/*`.

Документация эндпоинтов появится после Шага 4.

## Примечания

- Координаты точек хранятся в нормализованном виде (0..1) относительно
  размеров фото. Это защищает от рассинхронизации при ресайзе.
- Калибровка обязательна перед началом разметки.
- Все измерения вычисляются на сервере, не на клиенте.
